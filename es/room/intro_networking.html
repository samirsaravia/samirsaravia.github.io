T 2 The OSI Model

Preguntas y respuestas

¿Qué nivel elegiría para enviar datos: TCP o UDP?

4

nivel 4 Transporte (Transport)
Los dos protocolos más utilizados en esta nivel son TCP (Protocolo de control de transmisión) y UDP (Protocolo de datagramas de usuario).

¿Qué nivel verifica los paquetes entrantes para asegurarse de que no estén dañados?

2

Nivel 2 Data Link
Una de las funciones importantes al recibir datos en este nivel es verificar los datos recibidos para asegurarse de que no se hayan corrompido durante la transmisión, lo que puede suceder cuando los datos se envían desde el nivel 1: nivel física

¿En qué nivel se podrían formatear los datos preparándolos para transmitirlos?

2

NIvel 2 Data Link
Este nivel se centra en la transmisión de direcciones físicas y una de las tareas es presentar los datos en formatos adecuados para la transmisión.

¿Qué nivel es nivel de transmitir y recibir datos?

1

nivel 1 Física (Physical)
Un nivel bajo, justo en el hardware de la computadora, es donde se envían y reciben los pulsos eléctricos generados por los datos transmitidos a través de la red.

¿Qué nivel encripta, comprime o transforma los datos iniciales para crear un patrón?

6

nivel 6 Presentación (Presentation)
Este nivel recibe datos de el nivel de Aplicación que transforma los datos en un formato estándar.

¿Qué nivel rastrea la comunicación entre el servidor (host) y el cliente?

5

Nivel Sección(Session)
En este nivel se recibe datos correctamente formateados de el nivel 'Presentación' e intenta establecer una conexión con otros equipos a través de la Red. Gracias a este nivel es posible realizar varias solicitudes sin mezclar.

¿Qué nivel acepta solicitudes de comunicación de aplicaciones?

7

Nivel Aplicación (Application)

Según el modelo OSI, esta capa tiene opciones de red para programas que se ejecutan en una computadora, ofreciéndoles una interfaz para transmitir datos.

¿Qué nivel funciona con 'direccionamiento lógico'?

3

Nivel Red (Network)
Responsable de encontrar el destino del pedido o requisición. Se utiliza para ordenar la red correctamente. Actualmente, la forma más común de dirección lógica es el formato IPV4 (dirección común de los roteadores)

Al enviar datos a través de TCP, ¿cómo llamaría a los 'pequeños bits' que son fragmentos de datos?

Segmentos

Nivel Transporte
Esta nivel divide la transmisión en partes, usando TCP se les llama 'Segmentos' y en UDP se les llama 'Datagramas'.

¿Con qué capa se comunica el protocolo FTP?

7

Nivel Aplicación
proporciona opciones de red. FTP (protocolo de transferencia de archivos) se considera un protocolo de aplicación que mueve archivos entre un sistema local y uno remoto.

¿Qué protocolo de transporte se utiliza mejor para transmitir un "en vivo" (video en vivo)?

UDP

Con UDP
Los paquetes de datos están destinados a la velocidad en lugar de la precisión. Con TCP ocurre lo contrario, apto para enviar archivos o cargar Sitios; donde no se pueden perder datos en el camino.

T 4 El Modelo TCP/IP

Preguntas y respuestas

¿Qué modelo se introdujo primero, OSI o TCP / IP?

TCP/IP

EL modelo TCP/IP
Es similar al modelo OSI y sirve como base para las redes en la realidad actual.

¿Qué nivel del modelo TCP / IP cubre la funcionalidad de el nivel de transporte del modelo OSI? (nombre completo)

Transport

Los modelos TCP/IP y OSI
son diferentes, OSI tiene 7 niveles y TCP / IP tiene 4 niveles.

¿Qué nivel del modelo TCP / IP cubre la funcionalidad de el nivel 'Sesión' del modelo OSI? (nombre completo)

Application

En el modelo TCP/IP
El nivel "Aplicación" (aplicación) representa el equivalente a 3 niveles en el modelo OSI (Aplicación, Presentación, Sesión).

El nivel de interfaz de red del modelo TCP / IP cubre la funcionalidad de dos niveles en el modelo OSI, que es enlace de datos y ...?

Physical

El nivel Físico (Physical)
es el primer nivel, que está directamente en el hardware de la computadora.

¿Qué nivel del modelo TCP / IP funciona con la funcionalidad de el nivel OSI "Red"?

Internet

El nivel Internet
El modelo TCP / IP es responsable de encontrar el destino de su solicitud
(por ejemplo, facebook.com, google.com, etc.).

Que tipo de protocolo es TCP?

Connection-based

Connection-based o conexión basada.
significa que antes de enviar datos mediante TCP, el protocolo primero establece una conexión entre las computadoras (por ejemplo, su computadora y el servidor de Google) y luego envía los datos, pero este protocolo hace 3 tipos de conexión primero, para asegurarse de que esté conectado de todos modos.

Que significa SYN?

Synchronise

Synchronise o sincronización
Es el primer paso para verificar la conexión entre computadoras.

¿Cuál es el SEGUNDO paso de los 3 pasos?(3handshake)?

SYN/ACK

Para ter seguridad de la conexión el servidor (la segunda computadora)
responde con el mismo paquete de datos que se envió en el primer paso, agregando otro paquete (ACK) después de recibirlo y asegurándose de que ahora sea posible conectarse.

¿Cuál es la abreviatura del segmento "Reconocimiento" en el tercer paso de conexión?(3-way-handshake)

ACK

En esta etapa
El servidor recibe el paquete enviado en la segunda etapa del cliente y ahora garantiza la conexión y la transmisión de datos. (Es por eso que cuando buscas un sitio, toma un tiempo conectarse y muestra una imagen o un ícono "cargando")

T 8 Networking tools: Dig

Preguntas y respuestas

Cuando desea acceder a un sitio web, lo primero que hace su computadora es verificar la memoria caché para ver si la dirección ya está guardada, si no está en el caché, su computadora envía una solicitud a un servidor DNS recursivo. servidor de su proveedor de internet (ISP) y comprueba el caché, sino, la solicitud va a un servidor raíz, antes de 2004 había 13 servidores raíz en el mundo, hoy hay muchos más. Desde los servidores Root vuelve a los servidores TLD (.br / .bo / .uk) del respectivo país, luego pasa a los servidores Autoritativos, donde se encuentra el registro de la página que está buscando y permite la conexión al servidor del sitio web, todo en cuestión de milisegundos.

Que significa DNS?

Domain Name System
Wikipedia

¿Cuál es el primer tipo de servidor DNS que consulta su computadora cuando busca un dominio?

Recursive

¿Qué tipo de servidor DNS contiene el registro de un dominio específico (por ejemplo: .com / .com.bo, .edu.bo)?

¿Dónde se debe verificar el primer lugar al buscar la dirección de dominio?

Local Cache

En su propia computadora o dispositivo

Google trabaja en 2 servidores DNS. Uno de ellos tiene IP 8.8.8.8 ¿cuál es la dirección del otro?

8.8.4.4

Si una consulta de DNS tiene un TTL de 24 horas, ¿qué número mostrará la consulta DIG?

86400

TTL (Time To Live)
Cuando su computadora realiza una consulta por un nombre de dominio, guarda el resultado en el caché y muestra cuántos segundos es válido, por lo que 24 horas son 86400 segundos. (60 * 60 * 24)
